Wikipedia風ページ設計ガイド

基本ID: archive-wikipedia-style-guide / 最終更新: 2026-04-23 / 公開状態: Working Draft

Wikipediaらしさは、青いリンクや灰色の目次だけで決まるものではない。日本語版Wikipediaでは、導入部の定義文、節見出しの積み上げ方、脚注と出典の置き方、右寄せ情報ボックス、本文幅の制御が一体になって「百科事典のページらしさ」を形成している。Tenzu.jpではこの一体感を再現し、見た目だけでなく情報設計もWikipedia風に寄せる。

Wikipediaらしさの核

再現対象は「Wikipediaのブランド」ではなく、「百科事典の記事が読む前から信頼できそうに見える構造」である。優先順位は次の通り。

優先度 要素 理由
必須 冒頭の定義文、節構造、目次、脚注/出典、関連項目 本文の骨格そのもの。これが無いとWikipedia風ではなく単なる記事一覧に見える。
必須 本文の白背景、細い罫線、青リンク、左サイドナビ 閲覧時の第一印象を決める視覚言語で、既存のVector系レイアウトとも相性が良い。
あると強い 右寄せ情報ボックス、ハットノート、カテゴリ風ボックス 記事種別の違いを一瞬で伝えられ、Wikipediaらしい密度を出せる。
無くても許容 編集タブの実機能、履歴、利用者ページ、複雑なテンプレート挙動 Wikimedia固有機能であり、静的な資料サイトでは再現コストの割に効果が薄い。

ページ構成の定石

日本語版Wikipediaの多くの記事は、対象ジャンルが違っても次の順序に収束する。Tenzu.jpでも、この骨格を基本テンプレートとして採用する。

  1. ページタイトル
  2. 導入部: 対象を一文目で定義し、2段落目前後で重要事項を圧縮する
  3. 目次: 見出し数が増えるページでは導入部直後に置く
  4. 本文セクション群: 歴史、概要、特徴、関連制度、登場人物などを見出しで分割する
  5. 脚注: 注釈と出典を分けられる構造が望ましい
  6. 参考文献
  7. 関連項目
  8. 外部リンク

人物、作品、地理記事に共通する最小骨格は以下のとおり。

記事種別ごとの違い

記事種別 先頭付近で効く要素 本文で増えやすい節
人物 生没年、職業、代表作、家族関係の情報ボックス 来歴、人物像、作品、評価、関連人物
作品 公開年、媒体、作者、出演者、制作会社、興行成績 あらすじ、登場人物、制作、反響、受賞歴
地理 位置、面積、人口、行政区分、象徴、地図や旗画像 地理、歴史、政治、経済、交通、文化

Tenzu.jp向けの実装指針

  • 短いページでも、導入部と関連項目だけは切らない。
  • 長いページでは lead section、toc、references、related links を独立コンポーネント化する。
  • カテゴリ別に節名をある程度定型化し、人物/勢力/地理で見出しの癖を揃える。
  • カードUIは索引ページでは有効だが、記事ページ本文では多用しない。

UIで効く要素

Wikipedia風のUIは「装飾の多さ」ではなく「視線誘導の控えめさ」で成り立つ。余白、境界線、文字サイズ差を細かく効かせる方が、本物に近い印象になる。

要素 推奨 実装メモ
本文背景 白背景 + 薄い青灰の外枠 主役は常に本文。背景グラデーションは弱く、主張しすぎない。
本文幅 横に広すぎない可読幅 長文で目が滑らない幅を保ち、サイドバーとの役割差を明確にする。
リンク色 青を基本、訪問済みは紫系 本文内リンクが密にあってもノイズ化しない、Wikipediaらしい色設計。
見出し セリフ体 + 下罫線 本文テキストとの差を大きくしすぎない。静かな権威感が出る。
目次 薄灰背景 + 細枠 + 左寄せ階層 本文直後に置き、長文ページの地図として機能させる。
情報ボックス 右寄せ、淡色セル、細罫線 記事種別ごとの差分をもっとも効率よく出せる。
脚注番号 小さな上付きリンク 本文の邪魔をせず、後段の出典一覧へ自然に誘導する。

やりすぎるとWikipediaっぽくなくなる例

  • 大きなヒーロー画像や派手な背景を本文より目立たせる。
  • カード、影、角丸、グラデーションを記事本文に多用する。
  • 見出しをブランド見出しのように大きくしすぎる。
  • 本文よりメタ情報を先に見せすぎて、百科事典よりダッシュボードに寄る。

文体・リンク・出典ルール

導入文

  • 一文目で対象を定義する。例: 「Xは、Yである。」の構文を基本にする。
  • 導入部では評価より定義を優先し、何者か、どの時代か、なぜ重要かを短く示す。
  • リード文だけで記事全体の要旨が見える密度にする。

本文文体

  • 宣伝調、煽り調、感想調を避け、中立で説明的なトーンを保つ。
  • 断定しにくい内容は「とされる」「とみられる」「資料上は」のように根拠との距離感を明示する。
  • 箇条書きは補助として使い、説明の本体は段落で書く。
  • 一段落を短く保ち、話題転換は見出しか段落分割で処理する。

リンク設計

  • 初出の重要語には内部リンクを張るが、同一段落にリンクを詰め込みすぎない。
  • 記事をまたいで読ませたい語だけを選び、索引性の高い導線にする。
  • 関連項目は本文の補助線であり、単なるリンク集にしない。

出典と脚注

  • 本文で主張した事実に脚注を打ち、末尾に出典一覧を置く。
  • 注釈と出典を分けられる構造を持たせると、Wikipediaらしい整理感が出る。
  • 参考文献は「読むための資料」、脚注は「その記述の根拠」と役割を分ける。
  • 長い記事では、脚注なしの断定文が連続しないようにする。

Tenzu.jp実装チェックリスト

新しい記事テンプレートや既存ページ改修時は、次の項目を yes/no で確認する。

確認項目 判定

導入部だけで対象の重要性と位置づけが分かるかyes / no
見出しが階層化され、長文でも目次で走査できるかyes / no
記事種別に合った情報ボックスがあるか、または不要と判断できるかyes / no
本文の主要主張に脚注または出典が付いているかyes / no
参考文献、関連項目、外部リンクの役割が混ざっていないかyes / no
内部リンクが索引として機能しているかyes / no

参照した記事サンプル

観察対象として、少なくとも次の3系統の記事を基準にした。

  • 人物記事: 夏目漱石
  • 作品記事: 君の名は。
  • 地理記事: 東京都

これらは節構造がかなり異なる一方で、導入部、目次、情報ボックス、脚注、関連項目、外部リンクの置き方に強い共通性がある。
