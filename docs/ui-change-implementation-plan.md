# UI変更計画書 活用実装プラン

## 目的

`UI変更計画書.md`を、単なる方向性メモではなく、Tenzu.jp / 天通辞典を「実在する日本型ネット百科サービス風」に寄せるための実装バックログとして使う。最優先の判断基準は、トップページで世界設定を説明せず、検索・生活ニュース・急上昇ワード・掲示板・広告が混ざった入口に見せることである。

## 現状認識

- `index.html` はメインページ本文を大きく持っており、トップページ改修の主対象になる。
- `_layouts/default.html` は全ページ共通のヘッダー、検索、左ナビ、本文ラッパー、ページツール、フッターを管理している。
- `assets/app.js` は検索・ページ内目次などの既存挙動を担うため、リアルタイム更新を増やさず既存機能の接続だけ確認する。
- `assets/css/wiki.css` は現在のWiki風UIの主スタイルとして扱い、今回のポータル風・広告風・掲示板風のクラス追加先にする。
- `assets/images/` には既存ロゴや建物画像があるが、白い鳥マスコットと広告素材は未整備のため、まずはCSS/HTMLで成立する仮枠から始める。

## 実装方針

### 1. トップページを最初に変える

トップページはUI変更の印象を最も強く左右するため、最初の実装単位にする。

- 上部に「天通ポータル｜ニュース｜辞典｜動画｜生放送｜ゲーム｜占い｜ショッピング」のようなポータル導線を追加する。
- 検索欄を主役化し、カテゴリ選択つきの大きな検索フォームにする。
- 本文を「ようこそ」「最新ニュース」「注目の記事」「最近更新された記事」「新規作成記事」「人気カテゴリ」に分ける。
- 右カラムに「急上昇ワード」「広告」「掲示板で話題の記事」「新着記事」を置く。
- トップページから国家・政府・軍事・企業の重要設定語を直接目立たせる導線を減らす。
- 今日の日付、現在時刻、天気、運勢、リアルタイム閲覧数、速報中など、静的サイトと相性の悪い表現は入れない。

### 2. 共通レイアウトを段階的に寄せる

トップページの専用UIだけで完結させず、記事ページにも展開できる共通部品として整理する。

- `_layouts/default.html` のヘッダーを、日本型ポータル導線と検索に対応できる構造へ分ける。
- 左メニューは「メニュー」「記事の種類」「参加・支援」に再編し、百科サイトらしい真面目な導線として残す。
- 記事ページの右側にはページツールだけでなく、広告・急上昇ワード・新着記事を表示できる補助カラムを用意する。
- フッターの「作中世界の資料網」というメタ説明は、トップページの実在サイト風演出と衝突しやすいため、利用規約・ヘルプ・運営情報風の表現へ置換する。

### 3. 記事ページをWikipedia寄りに整える

トップページの雑然さに対して、記事ページは読みやすい百科記事として見せる。

- 記事タイトル直下に「出典: 天通辞典」を表示する。
- 「記事 / ノート」「閲覧 / 編集 / 履歴表示」のタブ風UIを追加する。
- 既存の目次生成と競合しないよう、本文内の目次・テンプレート・infobox表示をCSSで整える。
- 注意テンプレート、脚注、関連項目、カテゴリ、掲示板過去ログを記事下部の標準ブロックとして追加できるようにする。
- 掲示板は投稿機能を作らず、静的な過去ログ表示に限定する。

### 4. 広告・掲示板・急上昇ワードを演出部品化する

サイトの実在感を出す部品は、記事内容と混ぜすぎず、再利用可能なUIとして作る。

- 広告は架空サービス名だけを使い、`[PR]` または `[広告]` 表記、強いCTA、黄色・赤・緑のボタンで安っぽさを出す。
- 実在企業名や実在サービス名は広告に使わない。
- 急上昇ワードは重要設定語ではなく、ネットミーム・俗語・くだらない記事名を中心にする。
- 掲示板で話題の記事は、議論の正しさよりもコメント数と雑談感を優先する。
- 画像素材がなくても初期実装できるよう、広告はテキスト広告・CSS装飾から開始する。

## 優先順位つき作業バックログ

### Phase 1: トップページの方向転換

1. `index.html` のトップ本文から、設定資料集に見える紹介・主要勢力前面の導線を整理する。
2. `index.html` にポータル導線、ようこそ欄、生活ニュース、急上昇ワード、掲示板で話題の記事、広告枠を追加する。
3. `assets/css/wiki.css` に `.portal-bar`, `.site-header`, `.page-layout`, `.left-column`, `.main-column`, `.right-column`, `.box`, `.ad-box`, `.word-list`, `.news-list`, `.bbs` を追加する。
4. 既存検索が壊れないよう、検索フォームのIDや `assets/app.js` との接続を確認する。

### Phase 2: 共通ナビとレスポンシブ

1. `_layouts/default.html` の左メニューを百科サイト風の分類に変更する。
2. トップページと記事ページで同じ左メニューを使えるよう、トップページ専用条件に依存しすぎない構造へ整える。
3. 右カラムの広告・補助情報を記事ページでも使えるようにする。
4. スマホでは「ヘッダー、検索欄、メイン本文、右カラム、左メニュー」の順に縦積みにする。

### Phase 3: 記事ページの百科化

1. `_layouts/default.html` に記事タブ風UIを追加する。
2. `assets/css/wiki.css` に `.article-tabs`, `.notice-box`, `.infobox`, `.toc`, `.categories`, `.bbs`, `.comment` の見た目を追加する。
3. 代表記事を1〜2本選び、注意テンプレート・infobox・脚注・関連項目・カテゴリ・掲示板過去ログを試験導入する。
4. 既存のページ内目次生成が新しい本文構造で正しく動くか確認する。

### Phase 4: 素材と演出の仕上げ

1. 白い鳥型マスコットを `assets/images/` に追加し、トップページのようこそ欄へ配置する。
2. 広告素材を追加する場合は、実写素材風・商品写真風・ゲーム広告風に寄せ、AI絵っぽさを抑える。
3. 広告文言、急上昇ワード、掲示板コメントのトーンを増やし、サイト全体の生活ノイズを強める。
4. 実装後にスクリーンショットを取り、トップページが「架空世界の設定資料集」ではなく「日本型ネット辞典の入口」に見えるか確認する。

## 完了条件

- トップページの主役が世界設定の説明ではなく、検索・ニュース・急上昇・掲示板・広告になっている。
- トップページに政治・軍事・政府・企業の核心語が過剰に出ていない。
- 日付・時刻・天気・運勢・リアルタイム閲覧数など、静的サイトで古びる要素がない。
- 記事ページはWikipedia風の本文構造を維持しつつ、下部掲示板と右広告でニコニコ大百科風の雑味が出ている。
- 広告はすべて架空名で、実在企業・実在サービスに誤認されない。
- PCで情報密度が高く、スマホでは読める順序に縦積みされる。

## 推奨する最初のPR

最初のPRでは、範囲をトップページとCSSに絞る。

- `index.html` のトップページ構成を三カラムのポータル風に変更する。
- `assets/css/wiki.css` にトップページ用のボックス、右カラム、広告、ニュース、急上昇ワードのスタイルを追加する。
- `_layouts/default.html` は最小限の調整に留め、既存の記事ページを大きく壊さない。
- マスコット画像は後続PRに回し、初回はCSSの仮マスコットまたはプレースホルダーで進める。
