# Tenzu Editor Skill

## 優先順位

本スキルは以下の2層で判定する。

- **Tier-1（既存ルール / 常に優先）**
  1. 天通辞典の本文は中立・説明調で記述し、宣伝調/煽り調/感想調を避ける。
  2. 導入部は「対象の定義」を最初に置き、本文は見出しで階層化する。
  3. Wikipedia風の情報設計（導入・目次・本文節・脚注/出典・関連項目）を維持する。
  4. 主要主張には根拠を添え、出典と注釈の役割を混同しない。
  5. UIより本文可読性を優先し、百科事典として自然な読書順を崩さない。

- **Tier-2（今回追加仕様 / Tier-1と非競合な範囲で適用）**
  1. 架空設定の明示ハットノート、メタ視点節、出典方針の追加運用を許可する。
  2. ただし、Tier-1の中立性・構造整合・可読性を壊す実装は不採用とする。
  3. 「導入可能か」は禁則語・記事構造・出典粒度の3点で判定する。

## 競合しやすい項目の対比（Tenzu許可/禁止）

| 項目 | Tenzuで許可 | Tenzuで禁止 |
|---|---|---|
| 架空明示ハットノート | 導入直下で「本記事は架空設定を含む」等を簡潔に明示する。 | 断定的な免責文を長文化し、本文より目立たせる。 |
| メタ視点節 | 末尾付近に「編集上の注記」「設定上の留意点」を短く置く。 | 冒頭から制作裏話・執筆都合を展開し、主題説明を圧迫する。 |
| 出典方針 | 作中一次資料 / 設定資料 / 外部資料の粒度を分けて表示する。 | 「出典なし」断定文の連続、または参考文献と脚注の混同。 |

## 「被らない範囲」の具体例

Tier-2の導入は、以下のようにTier-1と役割が被らない範囲で行う。

1. **導入可**: UI構造（ハットノート位置・注記ボックス）を追加する。  
   **導入不可**: 禁止語ルールを上書きして文体を崩す。
2. **導入可**: 既存見出しを保ったまま「メタ視点節」を末尾に増設する。  
   **導入不可**: 「概要」「歴史」など基幹節を削って置換する。
3. **導入可**: 出典ラベルに「作中資料」「外部資料」を併記する。  
   **導入不可**: すべてを「参考」扱いにして根拠対応を曖昧化する。
4. **導入可**: 架空明示を1〜2文の補助説明に留める。  
   **導入不可**: 免責文を長文化してリード本文を押し下げる。
5. **導入可**: 既存内部リンク導線を維持した注記追加。  
   **導入不可**: 注記追加のために重要語リンクを大量削除する。

## 統合後の判定フロー（短手順）

1. **入力確認**: 依頼内容を「記事本文変更」「構造変更」「出典変更」に分解する。  
2. **禁則チェック**: 禁止語・過剰免責・煽り表現がないか確認する。  
3. **記事構造チェック**: 導入→目次→本文節→関連項目の骨格を維持できているか判定する。  
4. **出典粒度チェック**: 主張と根拠の対応、注釈/出典/参考の区別が保たれているか確認する。  
5. **最終適用判定**: Tier-1適合を必須条件とし、満たす場合のみTier-2追加仕様を反映する。
