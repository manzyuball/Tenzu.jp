# Tenzu Editor

このスキルでは、必要になった時だけ `references/` を読む。

## 参照ルール（いつ読むか）
- レイアウトやUIコンポーネント（SiteShell〜Navbox）を決めるとき: `references/wiki-like-layout.md`
- 第一文、導入部、節構成、日本語表記を整えるとき: `references/writing-style-ja.md`
- 完成度ランク判定、チェック、受け入れ可否を判断するとき: `references/quality-model.md`

上記以外の詳細は `SKILL.md` に重複して書かない。

# Tenzu Editor Skill

## 優先順位

本スキルは以下の2層で判定する。

- **Tier-1（既存ルール / 常に優先）**
  1. 天通辞典の本文は中立・説明調で記述し、宣伝調/煽り調/感想調を避ける。
  2. 導入部は「対象の定義」を最初に置き、本文は見出しで階層化する。
  3. Wikipedia風の情報設計（導入・目次・本文節・脚注/出典・関連項目）を維持する。
  4. 主要主張には根拠を添え、出典と注釈の機能を混同しない。
  5. UIより本文可読性を優先し、百科事典として自然な読書順を崩さない。

- **Tier-2（基準日仕様 / Tier-1と非競合な範囲で適用）**
  1. 公開記事は2026年5月6日時点の天通辞典として書く。
  2. 本編開始後の出来事、未発生事件、未来から見た評価、制作上の説明は不採用とする。
  3. 「導入可能か」は基準日、禁則語、記事構造、出典粒度の4点で判定する。

## 競合しやすい項目の対比（Tenzu許可/禁止）

| 項目 | Tenzuで許可 | Tenzuで禁止 |
|---|---|---|
| 基準日 | 2026年5月6日までに公表・観測・記録された事項を書く。 | 未発生事件、未来から見た評価、後日の結果を書く。 |
| 文体 | 公表資料、報道、企業資料、証言、推計として距離を取る。 | 「本作」「物語」「設定」「世界観」など制作上の語で説明する。 |
| 出典方針 | 公的資料 / 天通系列資料 / 外交記録 / 業界資料 / 証言 / 推計を分けて表示する。 | 「出典なし」断定文の連続、または参考文献と脚注の混同。 |

## 「被らない範囲」の具体例

Tier-2の導入は、以下のようにTier-1と機能が被らない範囲で行う。

1. **導入可**: 2026年5月6日時点の公表資料として書き換える。
   **導入不可**: 発生していない事件を記事化する。
2. **導入可**: 「とされる」「報じられている」など資料距離を取る。
   **導入不可**: 未来から見た因果や結果を断定する。
3. **導入可**: 出典ラベルに資料区分を併記する。
   **導入不可**: 制作資料・読者向け解説として本文を書く。

## 統合後の判定フロー（短手順）

1. **入力確認**: 依頼内容を「記事本文変更」「構造変更」「出典変更」に分解する。  
2. **基準日チェック**: 2026年5月6日以後の出来事や未来視点がないか確認する。
3. **禁則チェック**: 禁止語・過剰免責・煽り表現がないか確認する。
4. **記事構造チェック**: 導入→目次→本文節→関連項目の骨格を維持できているか判定する。
5. **出典粒度チェック**: 主張と根拠の対応、注釈/出典/参考の区別が保たれているか確認する。
6. **最終適用判定**: Tier-1適合を必須条件とし、満たす場合のみTier-2追加仕様を反映する。
