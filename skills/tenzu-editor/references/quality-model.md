# 品質モデル

## 完成度ランク

### A（公開可）
- 導入部、目次、本文、脚注/出典、関連項目、外部リンクが整備されている。
- 主要主張に根拠が付き、読書順が自然である。
- UIが本文可読性を阻害しない。

### B（改稿中だが運用可）
- 骨格（導入部・本文見出し・関連導線）は成立している。
- 一部の出典不足や節粒度の粗さが残る。
- 追記計画が明確で、破綻した断片がない。

### C（下書き）
- 定義文または節構造が未整備で、要旨把握が難しい。
- 出典・導線・UI整合の不足が目立つ。
- 公開前に大幅改稿を要する。

## チェックリスト
- 第一文で対象を定義している。
- 導入部だけで対象の重要性と位置づけが伝わる。
- 見出し階層が成立し、目次で走査できる。
- 記事種別に応じたInfoboxの要否判断ができている。
- 主要主張に脚注または出典がある。
- 参考文献・関連項目・外部リンクの役割が混在していない。
- 内部リンクが索引導線として機能している。
- UI装飾が本文可読性を上回っていない。
- ページ上から下まで読書順が自然である。

## 受け入れ条件
- 最低条件はランクB以上。
- 新規公開は、定義文・導入部・本文見出し・関連導線が必須。
- ランクAを目指す場合、主要主張への根拠付与と終端要素（脚注/参考文献/関連項目/外部リンク）を完備する。
- ランクCは原則公開しない（作業ブランチまたは下書き管理に限定）。
