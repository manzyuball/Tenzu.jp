# Wikipediaライク・レイアウト仕様

## 対象範囲
この文書は、Tenzu記事ページのレイアウトとUIコンポーネント仕様（SiteShell〜Navbox）を定義する。

## レイアウト全体（SiteShell）
- 本文を中心にし、白背景と細い境界線で静かな情報密度を作る。
- 横幅は長文可読性を優先し、広げすぎない。
- サイドナビと本文の視覚的機能を分離する。
- 派手な背景・強いグラデーション・大きすぎるヒーロー表現は避ける。

## UIコンポーネント仕様

### 1) LeadSection（導入部）
- 記事冒頭に配置する。
- 1文目で対象を定義し、続く段落で重要事項を圧縮する。
- 導入部のみで記事全体の要旨が把握できる密度を目標にする。

### 2) TOC（目次）
- 導入部直後に配置する。
- 薄灰背景・細枠・左寄せ階層で構成する。
- 見出しが増えるページの走査性を担保する。

### 3) ContentSections（本文セクション群）
- 見出し階層で話題を分割し、段落中心で説明する。
- 典型順序は「概要/来歴/構成/特徴」→詳細節。
- 記事種別ごとに節名をある程度定型化する。

### 4) Infobox（情報ボックス）
- 右寄せ配置、淡色セル、細罫線を基本とする。
- 人物・作品・地理など記事種別差分を先頭付近で提示する。
- 不要な記事では無理に設置しない。

### 5) ReferencesBlock（脚注・参考文献）
- 本文の主張に対応する脚注を配置する。
- 注釈と出典を分離できる構造を推奨する。
- 参考文献は「読むための資料」、脚注は「記述の根拠」として分ける。

### 6) RelatedLinks（関連項目）
- 本文の補助線として機能する内部導線を置く。
- 単なるリンク集にしない。

### 7) ExternalLinks（外部リンク）
- 関連項目の後段に配置する。
- 参照価値の高い外部情報のみを掲載する。

### 8) Navbox（遷移ボックス）
- 記事末尾に配置し、周辺記事の回遊を補助する。
- 本文可読性を阻害しない簡素なデザインを採用する。

## 見た目の禁則
- 角丸・影・カード演出を本文領域へ多用しない。
- 見出しをブランドUIのように過大化しない。
- メタ情報を本文より先に目立たせすぎない。
